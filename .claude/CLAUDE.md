# Krishna Shukla — Engineering Platform (kr1shna.xyz)

This is **Krishna Shukla's personal engineering platform**: a built, deployed Next.js 14 +
TypeScript + Tailwind site (framer-motion, three.js), auto-deployed to Vercel at `kr1shna.xyz` on
every push to GitHub `BrainTeaser1/Website`. It is **past the design phase** — the UI, motion,
architecture gallery, and navigation are mature. Future work prioritizes **content depth**
(richer architecture writeups, build logs, deep-dives) and **discoverability/performance** over
visual redesigns.

Project guidance is split into component rules under `.claude/rules/`.
Each rule is imported below; edit the individual files to change behavior.

## Modes

- @rules/recreation-workflow.md — pixel-accurate recreation from a reference image (legacy; the site is now a built Next.js app, not single-HTML recreation).
- @rules/portfolio-mode.md — designing an original personal/brand website.

## Design

- @rules/design-philosophy.md — how the result should feel; what to avoid.
- @rules/visual-direction.md — preferred style and reference aesthetics.
- @rules/personal-branding.md — how to present the user.

## Content & Implementation

- @rules/project-presentation.md — projects as case studies.
- @rules/technical-standards.md — required techniques and stack.
- @rules/technical-defaults.md — default technical assumptions (current Next.js stack).
- @rules/performance.md — performance is a hard requirement; specific traps to avoid.
